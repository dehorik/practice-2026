# Руководство по запуску сайта

Этот документ описывает запуск сайта проекта NeoPixel на локальной машине: от клонирования репозитория до открытия страницы в браузере. Сайт находится в директории `site` и реализован как статический многостраничный сайт с использованием Vite в качестве dev-сервера и инструмента сборки.

## Предварительные требования

Для запуска понадобятся:

- установленный Git;
- установленный Node.js;
- установленный npm;
- доступ к терминалу;
- браузер.

Проверить наличие инструментов можно командами:

```bash
git --version
node --version
npm --version
```

## Клонирование репозитория

Сначала нужно получить локальную копию проекта:

```bash
git clone https://github.com/dehorik/practice-2026
```

После клонирования нужно перейти в директорию репозитория:

```bash
cd practice-2026
```

Сайт расположен во вложенной директории `site`, поэтому далее все команды запуска выполняются из нее:

```bash
cd site
```

## Установка зависимостей

Перед первым запуском необходимо установить npm-зависимости:

```bash
npm install
```

Эта команда скачивает Vite и служебные пакеты, необходимые для локального запуска и сборки сайта. После установки появится директория `node_modules`; она не предназначена для ручного редактирования и обычно не добавляется в Git.

## Запуск локального сервера

Для запуска сайта используется команда:

```bash
npm run dev
```

После успешного запуска Vite выведет адрес локального сервера. Обычно это:

```text
http://localhost:5173/
```

Если порт `5173` уже занят, Vite автоматически выберет следующий свободный порт, например:

```text
http://localhost:5174/
```

Нужно открыть показанный адрес в браузере. После этого должна загрузиться домашняя страница сайта NeoPixel. Далее можно нажимать кнопки и переходить между разделами: «О проекте», «Участники», «Журнал» и «Ресурсы».

## Проверка перед публикацией

Для проверки производственной сборки можно выполнить:

```bash
npm run build
```

Команда создает директорию `dist`, в которую Vite помещает оптимизированную версию сайта. Если сборка завершилась без ошибок, значит структура страниц, подключение CSS, JavaScript и локальных ресурсов корректны с точки зрения сборщика.

При необходимости можно локально посмотреть собранную версию:

```bash
npm run preview
```

Vite снова покажет локальный адрес, который нужно открыть в браузере.

## Остановка сервера

Чтобы остановить локальный сервер, нужно вернуться в терминал, где выполняется `npm run dev`, и нажать:

```text
Ctrl+C
```

После этого сайт перестанет быть доступен по локальному адресу.

## Бонус: публикация через ngrok

ngrok позволяет временно открыть локальный сайт во внешний интернет. Это удобно, если нужно быстро показать проект преподавателю, куратору или участникам команды без полноценного размещения на хостинге.

Сначала запускается сайт:

```bash
cd practice-2026/site
npm run dev
```

Нужно запомнить порт, который показал Vite. Например:

```text
http://localhost:5173/
```

Затем во втором терминале запускается ngrok на этот же порт:

```bash
ngrok http 5173
```

Если Vite запустился на `5174`, команда должна быть такой:

```bash
ngrok http 5174
```

После запуска ngrok выдаст публичную ссылку вида:

```text
https://example-name.ngrok-free.app
```

Именно эту ссылку можно отправить другим людям. Пока работают Vite и ngrok, сайт будет доступен по внешнему адресу.

### Возможные особенности ngrok

На бесплатном тарифе ngrok может показывать промежуточное предупреждение безопасности. Это нормальное поведение сервиса. Чтобы перейти на сайт, посетителю достаточно нажать кнопку `Visit Site`.

Также Vite может заблокировать запрос с внешнего ngrok-домена сообщением о `server.allowedHosts`. В конфигурации сайта уже предусмотрено разрешение для доменов:

```js
allowedHosts: [".ngrok-free.dev", ".ngrok-free.app"]
```

Если используется другой домен, его нужно добавить в `site/vite.config.js`.
