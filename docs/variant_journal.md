# Журнал выполнения вариативной части

Тема: `Python: A Simple Web Server`.

## Выполненные шаги

1. Изучена постановка вариативной части практики и выбран раздел `Build your own Web Server` из репозитория `codecrafters-io/build-your-own-x`.
2. В качестве основной технологии выбран материал `Python: A Simple Web Server`, посвященный созданию простого веб-сервера и обработке HTTP-запросов.
3. Изучены базовые понятия предметной области: TCP-сокеты, IP-адрес, порт, HTTP-запрос, HTTP-ответ, заголовки, тело ответа, коды состояния.
4. Сформулированы требования к учебной реализации: работа без веб-фреймворков, поддержка `GET` и `HEAD`, отдача статических файлов, корректные ошибки, защита от выхода за пределы директории.
5. Спроектирована структура приложения в директории `src/`: основной файл сервера, директория публичных файлов, демонстрационные HTML/CSS-страницы и тестовая директория для листинга.
6. Реализован запуск TCP-сервера через стандартный модуль `socket`.
7. Реализовано чтение HTTP-запроса из клиентского соединения и ограничение максимального размера запроса.
8. Реализован разбор стартовой строки HTTP, заголовков и query-параметров.
9. Реализована маршрутизация запросов: статические файлы, `/api/time`, `/api/echo`.
10. Реализована выдача `index.html` для директорий и HTML-листинг директории при отсутствии индексного файла.
11. Добавлена защита от path traversal через нормализацию и проверку итогового пути.
12. Добавлена творческая модификация: JSON API, которое показывает текущее UTC-время и отражает параметры запроса.
13. Проведена проверка компиляции Python-файла командой `python3 -m py_compile`.
14. Сервер был запущен локально на `127.0.0.1:8091`.
15. Проверены сценарии: главная страница, JSON API, `HEAD`-запрос, отсутствующий файл с ответом `404`.
16. Обновлен корневой `README.md` с кратким описанием вариативной части.
17. Подготовлено подробное руководство `docs/variant_task.md`.
18. Подготовлен данный журнал выполнения работ.
