# Документация проекта

Директория `docs` содержит вспомогательные материалы по учебному проекту, связанные с сайтом, выполненной индивидуальной задачей и общей навигацией по документации. Файлы предназначены для быстрого понимания структуры проекта без необходимости просматривать исходный код или восстанавливать ход работы по отдельным коммитам.

## Состав документации

### `site.md`

Руководство по запуску статического сайта проекта на локальной машине. В документе описан путь от клонирования репозитория до открытия сайта в браузере, включая установку зависимостей, запуск Vite dev-сервера и проверку результата. В качестве дополнительного сценария приведена публикация локального сайта во внешний интернет через ngrok.

### `neopixel_task.md`

Подробное описание индивидуальной задачи «Реализация базового управления Z-осью» в рамках проекта NeoPixel. Документ раскрывает инженерный смысл задачи, ее место в архитектуре малого фотополимерного 3D-принтера, используемые аппаратные компоненты, программную логику и учебную ценность выполненной работы.

### `variant_task.md`

Подробное техническое руководство по вариативной части практики: созданию простого HTTP-сервера на Python по теме `Python: A Simple Web Server` из репозитория `codecrafters-io/build-your-own-x`. Документ включает исследование предметной области, пошаговую реализацию, примеры кода, диаграммы, таблицы и описание творческой модификации.

### `variant_journal.md`

Краткий журнал выполненных работ по вариативной части. В нем перечислены основные этапы: выбор темы, исследование, проектирование, реализация, проверка и оформление документации.

## Назначение директории

Документация в этой директории выполняет несколько функций:

- фиксирует технический контекст проекта;
- упрощает запуск и демонстрацию сайта;
- отделяет содержательное описание работы от исходного кода;
- помогает представить результаты проектной деятельности в академической форме;
- служит опорной точкой для дальнейшего расширения материалов.

При добавлении новых документов рекомендуется обновлять этот файл, чтобы `docs/README.md` оставался центральной навигационной страницей по документации.
